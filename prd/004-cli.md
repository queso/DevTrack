# PRD 004: CLI Tool

## Summary

The DevTrack CLI. The base CLI is auto-generated from the OpenAPI spec by [swagger-jack](https://github.com/theaiteam-dev/swagger-jack), which produces a complete Go/Cobra project with all API endpoints as subcommands. On top of that generated base, we add repo-aware convenience commands. This is how hooks talk to the API, how you register projects, and how you query state from the terminal.

## Problem

The dashboard is great for the overview, but when you're in a terminal working on a project, you don't want to switch to a browser to check status or record an event. The CLI meets you where you are. It's also the transport layer that hooks use to push state to the API.

## Architecture

This is a **Go CLI project**, not Node.js. It lives in a `cli/` directory within the DevTrack repo.

Two layers:

1. **Auto-generated API commands** — swagger-jack reads the OpenAPI spec and generates a full Go/Cobra CLI project. This gives us every API endpoint as a subcommand with proper flags, auth, `--json` output, shell completions, and help text — all for free. These commands live under `devtrack <resource> <action>` (e.g., `devtrack projects list`, `devtrack events create`).

2. **Convenience commands** — hand-written Go commands added to the generated project using swagger-jack's custom code preservation (`swagger-jack:custom:start/end` blocks). These read `project.yaml`, infer context from git state, and compose the generated API commands internally. These are the human-friendly commands.

### swagger-jack workflow

```bash
# Install swagger-jack (Go tool)
go install github.com/theaiteam-dev/swagger-jack@latest

# Generate the base CLI from our OpenAPI spec
swaggerjack init --schema ./specs/openapi.yaml --name devtrack --output-dir ./cli

# Build it
cd cli && go build -o devtrack .

# All API commands work immediately
./devtrack projects list --json
./devtrack events create --project-id abc --type commit --json
./devtrack prs list --status open

# When the API spec changes, regenerate (preserves custom code blocks)
swaggerjack update --schema ../specs/openapi.yaml
```

### Generated project structure (from swagger-jack)

```
cli/
├── cmd/
│   ├── root.go              # root command, config loading, auth
│   ├── projects.go          # devtrack projects ...
│   ├── projects_list.go     # devtrack projects list
│   ├── projects_create.go   # devtrack projects create
│   ├── prds.go              # devtrack prds ...
│   ├── events.go            # devtrack events ...
│   ├── prs.go               # devtrack prs ...
│   └── ...                  # one file per endpoint
├── internal/
│   ├── client.go            # HTTP client with auth
│   ├── config.go            # config file + env var loading
│   ├── output.go            # JSON and table output formatting
│   └── errors.go            # error handling
├── main.go
└── go.mod
```

### Where convenience commands go

Convenience commands are added as new `.go` files in `cmd/` (e.g., `cmd/register.go`, `cmd/status.go`). They use swagger-jack's custom code preservation so they survive `swaggerjack update` regeneration. They call the generated `internal/client.go` HTTP client to hit the API.

## Convenience Commands

These commands are repo-aware. Run them from inside a tracked repo and they auto-detect context from `project.yaml` and git state.

### `devtrack register`

Register the current repo with DevTrack.

- Reads `project.yaml` from repo root (parse YAML with Go's `gopkg.in/yaml.v3`)
- Auto-detects `repo_url` from `git remote get-url origin`
- POSTs to `/api/v1/projects` using the generated HTTP client
- Validates manifest before sending
- Updates existing project if already registered
- Offers to install hooks

### `devtrack status`

Show current project status. Quick terminal check without leaving your editor.

```
$ devtrack status

  picking-app (sdlc · arcanelayer)
  ────────────────────────────────
  PRD:    Barcode Scanning Support (3/5 items done)
  Branch: feat/barcode-scanning
  PR:     #42 — open, checks passing
  Last:   2 hours ago
```

Note: this requires multiple API calls — project details, project status, active PRDs with work item counts, and open PRs filtered by current branch.

### `devtrack dashboard`

Cross-project overview from the terminal. The CLI equivalent of the home page.

```
$ devtrack dashboard

  12 projects · 3 need attention · 2 PRs open

  ● picking-app          Adding barcode scanning — 3/5 done     2h ago
  ● aiteam-brand         Landing page redesign — 5/5 done, PR ready   4h ago
  ○ k8s-infra            Migrate to Cilium — 1/3 done           2d ago
  ◌ arcanelayer-store    No active PRD                          14d ago
```

Legend: ● active today, ○ this week, ◌ stale

### `devtrack prs`

Show all open PRs across projects. Custom table formatting beyond what the generated `devtrack prs list` provides.

```
$ devtrack prs

  PR QUEUE (3 open)

  picking-app     #42  Add barcode scanning     open    2h
  aiteam-brand    #15  Landing page redesign     approved  4h
  k8s-infra       #8   Cilium network policies   draft   1d
```

### `devtrack event <type>`

Record an event. Primarily used by hooks, but available manually. Accepts hyphenated event names for human friendliness and maps them to the API's underscore-separated enum values.

```bash
devtrack event commit --branch main --hash abc123 --message "Fix picking bug"
devtrack event prd-updated --prd-id <id>
devtrack event session-start
devtrack event session-end
```

Mapping: `session-start` → `session_start`, `session-end` → `session_end`, `prd-updated` → `prd_updated`.

### `devtrack sync`

Force a full sync of the current repo's state to the API:
- Re-reads `project.yaml` and updates project
- Scans `prd_path` and syncs all PRDs + work items
- Scans `content_path`/`draft_path` and syncs content items
- Fetches open PRs from GitHub and syncs

Useful for initial setup or if hooks missed something.

### `devtrack ideas`

Content workflow shortcut. Manage the idea backlog.

```bash
devtrack ideas                          # list ideas for current project
devtrack ideas --all                    # list ideas across all content projects
devtrack ideas add "Why PRDs matter"    # add an idea
devtrack ideas add "Solo dev workflow" --tags blog,ai --summary "How I manage 16 repos"
devtrack ideas promote <id>             # promote idea to draft (scaffolds file in repo)
```

### `devtrack hooks install`

Install Claude Code hooks for the current repo.
- Reads `project.yaml` to determine workflow type
- Writes hook scripts to `.claude/hooks/`
- Hook scripts call `devtrack event` to push state to the API

### `devtrack hooks test`

Verify hooks are installed and can reach the API.

## Configuration

swagger-jack generates config loading in `internal/config.go`. The generated CLI supports config via file and env vars out of the box.

Global config at `~/.devtrack/config.yaml`:

```yaml
api_url: https://devtrack.owenslabs.com/api/v1
api_key: dt_xxxxxxxxxxxx
```

Environment variables override config file:
- `DEVTRACK_API_URL`
- `DEVTRACK_API_KEY`

## Generated Commands

These come for free from swagger-jack. Every API endpoint becomes a subcommand:

```bash
devtrack projects list
devtrack projects get <id>
devtrack projects create --name "foo" --workflow sdlc ...
devtrack prds list --project-id <id>
devtrack events list --project-id <id> --type commit
devtrack prs list --status open
# etc.
```

These mirror the API 1:1. The convenience commands above are sugar on top of these.

## Output Formatting

swagger-jack generates table output (default) and `--json` flag on every command. Convenience commands use the same `internal/output.go` formatting.

- Default: human-readable tables/summaries (like the examples above)
- `--json` flag on any command: raw JSON output for scripting
- `--quiet` flag: minimal output (just IDs or status codes) — added to convenience commands
- Colors and formatting via terminal capabilities, gracefully degrade in non-TTY

## Error Handling

swagger-jack generates error handling in `internal/errors.go`. Convenience commands follow the same patterns:

- API unreachable: clear error message with suggestion to check config
- Invalid manifest: show validation errors with line numbers
- Auth failure: prompt to check API key
- All errors to stderr, never mixed with data output

## Build & Distribution

```bash
# Development
cd cli && go build -o devtrack .

# Install locally
cd cli && go install .

# Cross-compile for distribution
GOOS=linux GOARCH=amd64 go build -o devtrack-linux-amd64 .
GOOS=darwin GOARCH=arm64 go build -o devtrack-darwin-arm64 .
```

The binary is self-contained — no Node.js, no runtime dependencies.
